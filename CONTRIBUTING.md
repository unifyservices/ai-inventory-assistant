# Contributing to AI Inventory Assistant

Thank you for your interest in contributing! This document provides guidelines for contributing to this project.

## How to Contribute

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/your-feature`)
3. **Commit** your changes (`git commit -m 'Add: brief description'`)
4. **Push** to the branch (`git push origin feature/your-feature`)
5. **Open** a Pull Request

## Contribution Types

- Bug fixes and improvements
- New AI tools/capabilities
- UI enhancements
- Documentation updates
- Test coverage improvements

## Code Standards

- **Python**: Follow PEP 8 conventions
- **HTML/CSS/JS**: Use consistent formatting with the existing codebase
- All code must include inline comments for complex logic
- Type hints are encouraged for Python functions

## Commit Message Format

```
[Type]: Brief description

Types: Add, Fix, Update, Remove, Refactor, Docs
Example: Add: duplicate detection tool for inventory search
```

## Pull Request Process

1. Ensure all code is tested
2. Update README if adding new functionality
3. Request review from at least one maintainer
4. PRs are merged after approval

## Security

**CRITICAL**: Never include in any contribution:
- API keys, tokens, or credentials
- Client data or identifying information
- Azure subscription/tenant IDs

## Questions?

Open an issue or contact the maintainer.
